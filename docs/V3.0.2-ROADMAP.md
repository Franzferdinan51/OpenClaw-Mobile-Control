# DuckBot Go v3.0.2 Roadmap

**Status:** Planned after v3.0.1 stabilization  
**Updated:** 2026-03-11

## Goal
Ship a stabilization-and-polish release after the v3.0.1 chat routing fix.

## Priority Order

### P0 — Chat reliability
- Validate real OpenClaw request/response flow against a live gateway
- Add clearer send failure states and reconnect prompts
- Ensure message dedupe between optimistic UI and service-fed messages
- Confirm session routing uses the standard OpenClaw backend/session model

### P1 — Full bug sweep
- Remove remaining analyzer warnings in high-traffic screens/services
- Audit null-safety edge cases in dashboard, search, and settings flows
- Clean up stale backup/dev-only files that can confuse analysis/builds
- Verify first-launch/setup/autoconnect flow on a clean install

### P1 — UX polish
- Tighten chat empty/error/loading states
- Improve dashboard refresh/status messaging
- Refine actions/search handoff so navigation is obvious
- Review touch targets and feedback on core screens

### P2 — Tests and release hygiene
- Expand widget/integration coverage around chat and navigation wiring
- Add a release checklist for build/test/sync steps
- Keep version/changelog/status docs aligned with shipped builds

### P2 — Project sync discipline
- Keep `Android-App-DuckBot` as primary working tree until consolidation is complete
- Mirror validated changes into `DuckBot-Go-Project`
- Optionally reduce duplication by choosing one canonical repo path

## Definition of Done
- Chat works from all entry points
- Debug APK builds successfully
- Core tests pass
- Docs/changelog/version are current
- Both project folders match

# V3.0.5 Priorities

Working priority list based on the current bug backlog and the mobile app audit on March 11, 2026.

## Must-Have

### 1. Chat Stability
- Move to HTTP-first send/history with optional live updates.
- Reduce reconnect churn and false offline states.
- Keep the active session stable when navigating across chat, search, quick actions, and dashboard.
- Improve chat connection UX: connected, reconnecting, offline, and actionable failures.

### 2. Home Metrics That Are Actually Reliable
- Use the local metrics helper as the primary source for local runtime CPU, memory, uptime, and version.
- Show metric source and last refresh time clearly.
- Avoid mixing partial gateway payloads with helper-derived metrics in the same view.

### 3. Dashboard Cleanup
- Keep exactly one gateway/system health section.
- Make stale data obvious without noisy banners.
- Preserve quick access into agent detail and live activity views.

### 4. Local Runtime Productization
- Productize the local helper/service path.
- Add one-tap setup, repair, and diagnostics.
- Show status for gateway, metrics helper, Termux, and API install state.

### 5. Control Hub Coherence
- Add useful actions: restart gateway, open logs, backup now, refresh agents, test metrics, test chat, open helper status.
- Re-group actions into a real control surface instead of a flat list.

## Should-Have

### Agent Monitor
- Boss chat and broadcast polish.
- Activity feed with live state changes and tool activity.
- Better session drill-down and node/device controls.
- Stronger agent detail screen.

### Chat Nice-To-Haves
- Attachments.
- Voice input.
- Message persistence across restart.

### Docs and Release Reality
- Split shipped, in-progress, and planned work clearly.
- Document local runtime, metrics helper, and mobile-to-desktop connectivity.
- Make release notes match the real feature state.

## Later
- Mini office / office preview integration.
- Deeper BrowserOS workflow surfaces.
- Full analyzer cleanup, dead code cleanup, and stale backup removal.
- On-device smoke test automation for chat, search, quick actions, dashboard, and metrics.
